行为树与状态机类似，本质上只是在合适的时机和条件下调用**回调函数**的一种机制。这些回调函数内部执行什么操作，完全由自己设计。（可以理解为通过一个主序列来逐一调用函数体系库中的东西，每个调用都是独立的函数）

（PS：官方教程中交替使用“**to invoke the callback**”&#x548C;**“to tick”**&#x8FD9;两个表述，这里统一用tick）

在本教程中，我们的示例操作大多只是在控制台打印一些信息，实际项目中通常是复杂的决策树体系。

接下来，我们将创建这个简单的树：

## **如何创建自己的 ActionNodes？**

创建 TreeNode 的默认（且推荐）方法是通过继承。

```c++
// 这是一个自定义同步动作节点（SyncActionNode）的示例
// 该节点没有使用任何输入/输出端口（ports）
class ApproachObject : public BT::SyncActionNode
{
public:
  // 构造函数：接收节点名称，并调用基类 SyncActionNode 的构造函数
  // 第二个参数是一个空的端口列表，表示此节点不需要任何端口
  ApproachObject(const std::string& name) :
      BT::SyncActionNode(name, {})
  {}

  // 必须重写虚函数 tick()
  // 这是行为树执行到此节点时调用的核心函数
  // 同步节点的 tick() 会在同一个线程中阻塞执行，完成后立即返回状态
  BT::NodeStatus tick() override
  {
    // 输出当前节点的名称，用于调试或日志记录
    // this->name() 返回构造时传入的节点名称字符串
    std::cout << "ApproachObject: " << this->name() << std::endl;
    
    // 返回 SUCCESS 表示该动作成功执行完毕
    // 其他可能的返回值有 FAILURE 和 RUNNING
    return BT::NodeStatus::SUCCESS;
  }
};
```

另外，我们也可以使用**依赖注入**，根据函数指针（即“函子”）来创建一个 TreeNode。

*每个 TreeNode 实例都具有一个 `name`，只用于人类阅读，不需要确保唯一。*

> `tick()` 是实际执行操作的地方。它必须始终返回一个 NodeStatus，即 RUNNING、SUCCESS 或 FAILURE。



> **Functor**是一个跨学科术语，在计算机领域常译为"**仿函数**"或"**函数对象**"，具体含义取决于应用场景

functor 构建节点必须具有以下函数签名：

```plain&#x20;text
BT::NodeStatus myFunction(BT::TreeNode& self) 
```

例如：

```c++
// 引入 BehaviorTree.CPP 的命名空间，以便直接使用其中的类型和函数
// 如 NodeStatus、SyncActionNode 等，无需每次都写 BT::
using namespace BT;

// 这是一个简单的函数，返回 NodeStatus 类型
// 它模拟检查电池状态的操作，总是返回成功
// 该函数可直接用于 SimpleActionNode 或作为 BehaviorTree 中的条件/动作回调
BT::NodeStatus CheckBattery()
{
  // 输出电池状态信息，模拟检测过程
  std::cout << "[ Battery: OK ]" << std::endl;
  // 返回 SUCCESS，表示电池状态正常
  return BT::NodeStatus::SUCCESS;
}

// 夹爪接口类，封装了夹爪的打开和关闭操作
// 将函数包装成行为树中的 ActionNode
// 例如通过 BehaviorTree.CPP 的 Method 机制将 open() 和 close() 注册为节点
class GripperInterface
{
public:
  // 构造函数：初始化夹爪状态为打开（_open = true）
  GripperInterface(): _open(true) {}
  
  // 打开夹爪的操作
  // 返回 NodeStatus 以便与行为树框架直接兼容
  NodeStatus open() 
  {
    // 设置内部状态为打开
    _open = true;
    // 输出调试信息
    std::cout << "GripperInterface::open" << std::endl;
    // 返回 SUCCESS 表示操作成功完成
    return NodeStatus::SUCCESS;
  }

  // 关闭夹爪的操作
  NodeStatus close() 
  {
    // 输出调试信息
    std::cout << "GripperInterface::close" << std::endl;
    // 设置内部状态为关闭
    _open = false;
    // 返回 SUCCESS 表示操作成功完成
    return NodeStatus::SUCCESS;
  }

private:
  bool _open; // 共享信息，记录夹爪当前状态（true 为打开，false 为关闭）
  // 该变量被 open() 和 close() 修改，是类内的共享状态
};
```

这样，我们就可以使用以下任何一个函数来构建一个 `SimpleActionNode`：

* CheckBattery()

* GripperInterface::open()

* GripperInterface::close()

## **使用 XML 动态创建树**

让我们来看一下名为 **my\_tree.xml** 的以下 XML 文件：

```xml
 <root BTCPP_format="4" >
     <BehaviorTree ID="MainTree">
        <Sequence name="root_sequence">
            <CheckBattery   name="check_battery"/>
            <OpenGripper    name="open_gripper"/>
            <ApproachObject name="approach_object"/>
            <CloseGripper   name="close_gripper"/>
        </Sequence>
     </BehaviorTree>
 </root>
```

> 有关 XML 模式的更多详细信息，请参见[<u>此处</u>](https://www.behaviortree.dev/docs/learn-the-basics/xml_format)。



我们必须先将自定义的 TreeNodes 注册到 `BehaviorTreeFactory` 中，然后从文件或文本中加载 XML。

XML 中使用的标识符必须与注册 TreeNodes 时使用的标识符一致。

“name” 属性表示实例的名称（有没有无所谓）。

```c++
#include "behaviortree_cpp/bt_factory.h"
// 包含示例自定义节点的头文件（如 ApproachObject 等）
#include "dummy_nodes.h"
using namespace DummyNodes;

int main()
{
    // 创建行为树工厂对象，用于注册自定义节点和创建行为树实例
    BehaviorTreeFactory factory;

    // -------------------------------------------------------
    // 方式1：通过继承自 TreeNode 的类注册节点（推荐方式）
    // 将自定义类 ApproachObject 注册为节点类型 "ApproachObject"
    // 之后在 XML 文件中可以使用 <ApproachObject/> 标签
    factory.registerNodeType<ApproachObject>("ApproachObject");

    // -------------------------------------------------------
    // 方式2：使用函数指针注册简单条件节点（SimpleConditionNode）
    // 这里用 C++11 lambda 表达式将 CheckBattery 函数包装起来
    // lambda 捕获列表 [&] 表示以引用方式捕获外部变量（虽然此处未使用）
    // 参数 (TreeNode&) 是节点自身的引用，在 lambda 中未使用，但必须提供以匹配签名
    // 这种节点在 XML 中以 <CheckBattery/> 使用，执行时调用 CheckBattery() 并检查返回值
    factory.registerSimpleCondition("CheckBattery", 
        [&](TreeNode&) { return CheckBattery(); });

    // -------------------------------------------------------
    // 方式3：将类的成员函数注册为简单动作节点（SimpleActionNode）
    // 首先创建一个夹爪接口对象，它包含 open() 和 close() 两个动作
    GripperInterface gripper;
    
    // 注册打开夹爪的动作节点，名称为 "OpenGripper"
    // lambda 捕获 gripper 对象的引用，以便调用其 open() 方法
    factory.registerSimpleAction("OpenGripper", 
        [&](TreeNode&) { return gripper.open(); });
    
    // 注册关闭夹爪的动作节点，名称为 "CloseGripper"
    factory.registerSimpleAction("CloseGripper", 
        [&](TreeNode&) { return gripper.close(); });

    // 行为树通常在部署时（运行时）创建，但通常只在程序开始时创建一次
    // 这里从 XML 文件 "./my_tree.xml" 加载并构建行为树结构
    // createTreeFromFile 解析 XML 并实例化所有已注册的节点
    // 重要：当 "tree" 对象离开作用域时，所有树节点都会被销毁
    auto tree = factory.createTreeFromFile("./my_tree.xml");

    // 要执行行为树，需要不断 "tick" 它
    // tick 会根据树的逻辑（如 Sequence、Fallback 等）传播到子节点
    // 在这个例子中，整个 Sequence 的所有子节点都返回 SUCCESS，
    // 因此会按顺序全部执行
    // tickWhileRunning 会在一个循环中不断 tick 树，直到树返回 SUCCESS 或 FAILURE
    tree.tickWhileRunning();

    return 0;
}

/*
 * 预期输出：
 * [ Battery: OK ]
 * GripperInterface::open
 * ApproachObject: approach_object
 * GripperInterface::close
 */
```

**项目中可以将节点注册放在main.py中统一管理。**
